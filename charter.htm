Project Charter for SaaS based twitter api for all organisation

This document describes the scope, deliverables and main requirements for the "SaaS based twitter api for all organisation" project.

Revisions

Version Comment Who
July 24, 2014 Initial draft from email by Krishna Prasad David Hawley

Description

SaaS based api for all organization to get their twitter sentiment on live tweets. (needs clarification)

Scope

In scope:
  1. Design of API and construction, construction and testing of software realizing the API
  2. Deployment of the software to public cloud
Out of scope:
  • Performance testing, load testing
  • Anything to do with billing
  • User account management features

Deliverables

  1. Requirements documentation, at least covering the user profiles and the details of the use cases
  2. Documented API Design covering the major use cases
  3. Software which implements the design
  4. Automated acceptance test suite for the API
  5. Deployment on cloud

Key requirements (needs cleanup)

Functional requirements:
  1. We can classify the tweets as per their sentiment e.g: which tweets goes to “happiness", “anger", “just sharing or neutral” or we can classify in n ways as well.
  2. What is the mood of information i.e sentiment or as you said retrieve aggregate sentiment (for period of time)
  3. We can compare the sentiment/mood of two similar kind of companies.
  4. we could use the sentiment to advise the company to improve the quality of care of customer on the basis of their tweets.
Non-functional requirements:
  1. Based on open source software
  2. Supports multiple languages, e.g. JAVA, Python, PHP, GO (needs clarification)
  3. Both type of DB, NoSQL and MySQL (needs clarification)
